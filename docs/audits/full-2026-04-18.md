## Executive Summary

The ECC codebase continues on a strong upward trajectory since the April 9 audit. All 3,384 workspace tests pass (up from 2,569 with 1 failure in the prior audit). Architectural boundaries remain pristine -- zero dependency rule violations across layers. Clippy passes with zero warnings. The supply-chain audit (`deny.toml`) and security posture remain solid.

The most significant improvement is the resolution of the prior failing test (`sqlite_bypass_store::prune`). Test count has grown by 815 tests (+32%) in 9 days. The validate/patterns.rs extraction from the prior audit appears complete -- it was decomposed into focused submodules (frontmatter_validation.rs, code_block_scanning.rs, section_validation.rs, cross_ref_validation.rs).

Remaining concerns center on: (1) 13 files exceeding the 800-line project limit (up from 7, driven by continued feature growth), (2) 11 `SystemTime::now()` calls in `ecc-app` bypassing the `Clock` port (documented debt BL-133, only 2 out of 13 timestamp-dependent modules use the port), (3) bus factor remains maximum risk with a single contributor owning 100% of core Rust code, (4) extensive `let _ =` swallowed errors in cartography delta helpers (23 instances in one file), and (5) drift_check.rs contains stub TODO code that never executes its file-parsing logic.

The ecc-workflow crate continues to operate outside the hexagonal architecture, using direct `std::fs`, `std::process`, and `std::io` instead of port traits. This is architecturally intentional (standalone binary) but creates a quality asymmetry -- the crate has the densest concentration of oversized files (5 files > 500 lines).

## Cross-Domain Correlations

### [CORR-001] HIGH -- Bus Factor + Entire Codebase Hotspot

- **Severity**: HIGH (escalated from MEDIUM via bus-factor + hotspot rule)

- **Signal**: Single contributor owns 100% of 501 Rust source files across 2,832 commits. titouanlebocq: 2,426 commits (85.7%). All other contributors are community/fork activity on non-core files (agents, docs, configs). Every core file is a bus-factor risk.

### [CORR-002] MEDIUM -- Clock Port Bypass + Untestable Timestamps = Fragility

- **Severity**: MEDIUM
- **Sources**: Architecture + Error Handling + Testing
- **Signal**: 11 `SystemTime::now()` calls in `ecc-app/src/` bypass the `Clock` port trait. Only 2 of 13 timestamp-dependent modules use the proper port (`backlog.rs`, `workflow/recover.rs`). Known debt (BL-133) but scope has grown from 1 callsite to 11. Files affected: `hook/dispatch.rs`, `hook/handlers/tier2_tools/quality.rs`, `hook/handlers/tier3_session/mod.rs`, `hook/handlers/tier3_session/session_merge.rs`, `metrics_session.rs`, `claw/handlers/session.rs`, `worktree/mod.rs`, `bypass_mgmt.rs`, `metrics_mgmt.rs` (x2), `memory/crud.rs`.
- **Design Smell**: **Viscosity** -- using `SystemTime::now()` is easier than threading the Clock port through, making the wrong thing easier than the right thing.
- **Risk**: Tests cannot control time, making time-dependent behavior untestable. The prior audit's failing test was caused by exactly this pattern.
- **Remediation**: Add `Clock` to `HookPorts` and `MetricsPorts`. Replace all `SystemTime::now()` calls with port-based clock injection. This is a mechanical refactor that should be done in a single pass.

### [CORR-003] MEDIUM -- Oversized Files + High Churn = Rigidity Smell

- **Severity**: MEDIUM
- **Sources**: Evolution + Conventions + Architecture
- **Signal**: 13 non-test files exceed 800 lines (up from 7 in prior audit). The top 5 hotspots by change frequency are all oversized files. `phase_gate.rs` (959 lines, 25 changes), `transition.rs` (848 lines, 28 changes), `memory_write.rs` (677 lines, 20 changes) are all in `ecc-workflow` which lacks the hexagonal decomposition discipline of the main workspace.
- **Design Smell**: **Rigidity** -- large files with high churn resist change and make reviews harder.
- **Remediation**: Decompose `ecc-workflow` command files into focused submodules. Priority: `phase_gate.rs` (959 lines with 18 test functions inline).

### [CORR-004] MEDIUM -- Swallowed Errors + No Logging = Silent Failures

- **Severity**: MEDIUM
- **Sources**: Error Handling + Observability
- **Signal**: `cartography/delta_helpers.rs` has 23 `let _ =` error suppressions for filesystem operations (create_dir_all, write, rename, remove_file) and git commands. None of these failures are logged. If cartography scaffold creation fails silently, downstream features will see missing data with no diagnostic trail.
- **Risk**: Hard-to-debug failures in cartography pipeline. Violations cascade silently.
- **Remediation**: Replace `let _ =` with `if let Err(e) = ... { tracing::warn!(...) }` for all I/O operations. This is a mechanical fix.

### [CORR-005] LOW -- Stub Code + No Test = Dead Feature

- **Severity**: LOW
- **Sources**: Testing + Code Quality
- **Signal**: `drift_check.rs:35-36` contains `let expected_files: Vec<String> = Vec::new(); // TODO: parse from design` and `let actual_files: Vec<String> = Vec::new(); // TODO: parse from implement-done.md`. The drift check feature runs and produces output, but its file-level drift detection is a no-op. The TODO has been present since at least the prior audit.
- **Risk**: Users may trust drift reports that only check AC coverage, not file-level drift.
- **Remediation**: Either implement the file parsing or remove the dead code paths and document the limitation.

#### [SEC-003] `deny_unknown_fields` Missing on Deserialization Types
- **Severity**: MEDIUM
- **Location**: 38 `#[derive(Deserialize)]` types across crates
- **Principle**: Input validation at system boundaries (`rules/rust/security.md`)
- **Evidence**: Only 1 mention of `deny_unknown_fields` in the entire codebase, and it's a comment explaining why it's intentionally omitted on `WorkflowState`. The remaining 37 Deserialize types accept unknown fields silently.
- **Risk**: Typos in config files or YAML frontmatter are silently ignored. Users won't know their config key was misspelled.
- **Remediation**: Add `#[serde(deny_unknown_fields)]` to config types where forward compatibility is not required. Exclude types that explicitly need forward compat (like WorkflowState).

#### [SEC-004] SQL Parameterization
- **Severity**: MEDIUM
- **Location**: `ecc-infra/src/sqlite_bypass_store.rs`, `sqlite_cost_store.rs`, `sqlite_log_store.rs`, `sqlite_metrics_store.rs`
- **Evidence**: Most SQL queries use parameterized `?` bindings. Some use `format!()` for date arithmetic with `u64` inputs. Not exploitable (integer types prevent injection) but violates the convention of always using parameterized queries.
- **Remediation**: Replace `format!("datetime('now', '-{} days')", n)` with parameterized date arithmetic.

#### [SEC-005] Phase Gate Traversal Protection

- **Evidence**: `phase_gate.rs` includes `contains_encoded_traversal()` function that detects `%2F` and `%2E%2E` URL-encoded path traversal. Tests cover traversal attacks, absolute paths outside project, and worktree isolation.

## Top 5 Recommendations

1. **Decompose oversized files now.** `backlog.rs` at 1,467 lines is nearly double the limit. This is the highest-leverage change for maintainability. Split by use case: `next_id.rs`, `duplicates.rs`, `reindex.rs`, `list.rs`, `search.rs`.

2. **Resolve the Clock port debt (BL-133).** 11 `SystemTime::now()` calls bypass the established port pattern. This is the project's most visible architectural inconsistency and directly caused the prior audit's failing test. A single-pass refactor would eliminate the category.

3. **Log, don't swallow.** 23 silently swallowed errors in `delta_helpers.rs` plus scattered `let _ =` patterns across session handlers create invisible failure modes. Replace with `tracing::warn!` -- this is a 30-minute fix with outsized observability gains.

4. **Enable `#![warn(missing_docs)]` in `ecc-domain`.** The domain crate defines the vocabulary of the entire project. Documenting its types pays dividends across every crate that consumes them.

5. **Add `deny_unknown_fields` to config Deserialize types.** 37 types silently accept unknown fields. Users typing `tiemout: 30` instead of `timeout: 30` get no feedback. This is the lowest-effort, highest-user-impact change.
