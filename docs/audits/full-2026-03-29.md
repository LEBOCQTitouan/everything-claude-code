## Executive Summary

The ECC codebase is architecturally sound with clean hexagonal boundaries enforced at compile time. The Rust workspace structure is a textbook ports-and-adapters implementation with 6 port traits, 6 production adapters, and 6 test doubles. Clippy passes with zero warnings, and the 1742-test suite provides strong coverage for domain logic.

The primary risks cluster around **one critical hotspot** (`config/merge.rs` at 877 lines exceeding the 800-line limit), **silent failure paths in the install pipeline** (errors swallowed as warnings, no rollback on partial failure), and **documentation staleness** (test counts and crate counts self-contradict across docs). The typed hook merge path — the live production path — has zero unit test coverage, which is the single highest-priority finding.

Security posture is strong: no hardcoded secrets, proper stdin bounding, shell injection prevention via `Command::new` (no shell invocation), and atomic file writes. The main dependency risk is `serde_yml 0.0.12` with documented quality concerns.

## Cross-Domain Correlations

### [CORR-001] CRITICAL — Hotspot + Zero Test Coverage: `config/merge.rs` typed hook path

- **Severity**: CRITICAL
- **Sources**: [EVL-001] + [TEST-001] + [CODE-002]
- **Correlation**: The #1 evolution hotspot (`config/merge.rs`, 877 lines, 12 changes in 180 days) has zero test coverage for its typed hook merge functions (`merge_hooks_typed`, `remove_legacy_hooks_typed`) — the live production code path. Combined with the DRY violation (CODE-002: legacy detection duplicated 3 times), a regression in the typed path would be undetected and could corrupt `settings.json` hooks during install.
- **Evidence**: `merge_hooks_pure` has 10+ tests; `merge_hooks_typed` has 0. The typed path is called from `ecc-app/src/merge/mod.rs:382`. Legacy detection logic is copy-pasted between typed and untyped variants.
- **Risk**: A single-character typo in the typed legacy detection patterns would silently stop removing legacy hooks during install, accumulating stale hooks in user `settings.json`.
- **Remediation**: (1) Add tests for `merge_hooks_typed` mirroring the `merge_hooks_pure` suite. (2) Extract `is_legacy_command(cmd: &str) -> bool` shared between typed and untyped paths. (3) Split `merge.rs` into submodules to bring under 800 lines.

### [CORR-002] CRITICAL — Silent Install Failure + No Test Coverage: install pipeline error paths

- **Severity**: CRITICAL
- **Sources**: [ERR-001] + [ERR-002] + [ERR-003] + [TEST-003]
- **Correlation**: The install pipeline has three distinct silent failure modes (unreadable source dir → empty install, file read error → phantom change, hook merge failure → skipped hooks), and the install step functions (`steps.rs`, 219 lines) have zero unit tests. Integration tests only cover happy paths.
- **Evidence**: `step_hooks_and_settings` returns `()` — it cannot signal failure. `pre_scan_directory` swallows read errors. `InstallSummary.success` can be `true` even when hooks were not installed.
- **Risk**: A user runs `ecc install`, gets "Install complete", but has no hooks. The phase-gate, TDD enforcement, and dev toggle hooks — the core ECC value proposition — are silently absent.
- **Remediation**: (1) Change `step_hooks_and_settings` to return `Result`. (2) Propagate `pre_scan_directory` errors. (3) Add unit tests for install steps using in-memory FS doubles. (4) Test the error paths explicitly.

### [CORR-003] HIGH — Hotspot + Growing Fast + No Decomposition Plan: `config/validate.rs`

- **Severity**: HIGH
- **Sources**: [EVL-002] + [ARCH-006] + [CODE-001]
- **Correlation**: `config/validate.rs` grew 83% in 2 weeks (341 → 624 lines) and is the fastest-growing file. At current trajectory it will breach the 800-line limit within weeks. No decomposition plan exists.
- **Evidence**: 13 changes in 180 days, complexity density 0.090. Each new config feature adds more validation branches to this single file.
- **Remediation**: Extract per-domain validators behind a `Validate` trait before the file crosses 800 lines: `validate_hooks.rs`, `validate_agents.rs`, `validate_workflow.rs`.

### [CORR-004] HIGH — Convention Violations in CRITICAL Hotspot: `.unwrap()` + `Regex::new` in production

- **Severity**: HIGH
- **Sources**: [CONV-003] + [CONV-007] + [EVL-001]
- **Correlation**: The codebase has established `LazyLock<Regex>` as the convention (18 instances), but `detection/language.rs` has 6 `Regex::new().unwrap()` calls in production code. Combined with other `.unwrap()` calls in `flock_lock.rs` and `toggle.rs`, these represent panic risks in production paths.
- **Evidence**: `language.rs:89,92,118,142,143,190` — six call-time regex compilations with `.unwrap()`.
- **Remediation**: Promote to `LazyLock<Regex>` statics, matching the established pattern. Replace remaining `.unwrap()` calls in production with `?` propagation.

### [CORR-005] HIGH — Bus Factor 1 + Documentation Staleness: knowledge concentration risk

- **Severity**: HIGH
- **Sources**: [EVL-004] + [DOC-001] + [DOC-006] + [DOC-018]
- **Correlation**: The entire codebase has bus factor 1 (single contributor). Documentation — the primary knowledge transfer mechanism for a solo project — is stale and self-contradicting. Test counts are wrong in 4 places across 3 files, crate counts are wrong in 2 files, and CLAUDE.md contradicts itself.
- **Evidence**: CLAUDE.md says both "1698 tests" and "1562 tests" in the same file. ARCHITECTURE.md says "1404 tests" (24% drift). Actual: 1742.
- **Remediation**: (1) Fix all stale counts immediately. (2) Add a hook or CI gate that verifies test count in CLAUDE.md matches `cargo test` output. (3) Regenerate ARCHITECTURE.md and MODULE-SUMMARIES.md.

### [CORR-006] MEDIUM — serde_yml Risk + Domain Layer Coupling

- **Severity**: MEDIUM
- **Sources**: [SEC-001] + [ARCH-011]
- **Correlation**: `serde_yml` (supply chain risk, quality concerns) is used directly in `ecc-domain` — the most stable, most-depended-upon crate. A vulnerability in serde_yml would affect the domain layer, propagating risk to all upstream crates.
- **Evidence**: `ecc-domain/src/backlog/entry.rs:115` uses `serde_yml::from_str`. Already captured as BL-099 for migration.
- **Remediation**: Migrate to `serde-yaml-ng` (BL-099). Consider moving YAML parsing to `ecc-infra` behind a port trait.

### [CORR-007] MEDIUM — Verbose Flag Dead + Silent Install = No Debugging Path

- **Severity**: MEDIUM
- **Sources**: [OBS-002] + [ERR-001] + [ERR-003]
- **Correlation**: The `--verbose` flag has no effect (0 `log::info!` calls, 2 `log::debug!` calls). When install fails silently (swallowed errors, no rollback), there is no way to diagnose what happened. The user has no debugging path.
- **Evidence**: Running `ecc install --verbose` produces identical output to `ecc install`.
- **Remediation**: Add `log::info!` calls at install step boundaries and `log::debug!` for sub-steps. Ensure install errors flow to the error accumulator before they flow to verbose logs.
