# Changelog Generation

Generation skill for producing human-readable changelogs from git history. Parses conventional commits, groups by significance, and formats for the target audience.

## When to Activate

- During doc-generator's Step 4 (Generate Changelog)
- Before releases (summarise changes since last tag)
- When maintaining `CHANGELOG.md`
- When generating release notes

## Methodology

### 1. Input Sources

| Source | What it Provides |
|--------|-----------------|
| git-narrative skill | Structured commit data, decision trail, milestones |
| `git tag --list` | Version boundaries for grouping |
| `git log --format` | Raw commit messages for parsing |

### 2. Conventional Commit Parsing

Parse commit messages into structured entries:

```
<type>(<scope>): <description>

<optional body>

<optional footer>
```

Map types to changelog sections:

| Commit Type | Changelog Section | Audience |
|-------------|-------------------|----------|
| `feat:` | Added | Users |
| `fix:` | Fixed | Users |
| `perf:` | Performance | Users |
| `refactor:` | Changed | Developers |
| `docs:` | Documentation | Developers |
| `test:` | Testing | Developers |
| `chore:` | Maintenance | Developers |
| `ci:` | CI/CD | Developers |

### 3. Version Grouping

Group entries by version boundary:

**If semver tags exist:**
```markdown
## [1.2.0] - 2026-03-10

### Added
- feat: add dry-run mode to merge command (#45)

### Fixed
- fix: handle null input in parseConfig (#42)

## [1.1.0] - 2026-02-15
...
```

**If no tags exist**, group by time period:
```markdown
## Week of 2026-03-10

### Added
- Add dry-run mode to merge command

## Week of 2026-03-03
...
```

### 4. Entry Formatting

For each changelog entry:

1. **Rewrite for humans**: Convert terse commit messages to readable sentences
   - `fix: handle null input in parseConfig` → `Fixed crash when configuration file contains null values`
2. **Add context**: Include PR/issue numbers if present in commit footer
3. **Attribute**: Include author for external contributions
4. **Link**: Reference the commit SHA for traceability

### 5. Scope Control

| Scope | Behaviour |
|-------|-----------|
| Default | Last 100 commits or 6 months (whichever is smaller) |
| `--since=<tag>` | All commits since the specified tag |
| `--since=<date>` | All commits since the date |
| `--full` | Complete history (use sparingly) |

### 6. Breaking Changes

Highlight breaking changes prominently:

```markdown
## [2.0.0] - 2026-03-14

### BREAKING CHANGES
- **mergeDirectory()**: `strategy` parameter is now required (previously defaulted to 'overwrite')
- **Config format**: Migrated from JSON to YAML — run `npx migrate-config` to convert

### Added
...
```

Detection: Look for `BREAKING CHANGE:` in commit footer, or `!` after type (e.g., `feat!:`).

### 7. Filtering

Exclude from changelog:
- Merge commits (no informational value)
- `chore: update lock file` (noise)
- Revert + re-apply pairs (cancel out)
- Commits touching only test files (unless `--include-tests`)

## Output

- **File**: `docs/CHANGELOG.md`
- **Format**: [Keep a Changelog](https://keepachangelog.com/) compatible
- **Header**: `<!-- Generated by doc-generator | Date: YYYY-MM-DD -->`
