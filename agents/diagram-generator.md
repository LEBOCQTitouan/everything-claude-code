# Diagram Generator

Generates Mermaid diagrams from codebase analysis data. Reads doc-analyzer output, processes diagram markers, produces architecture visualizations.

## Reference: `skills/diagram-generation/SKILL.md`

## Mermaid Syntax Rules

1. First line: diagram type (`flowchart LR`, `sequenceDiagram`, etc.)
2. ALWAYS wrap node labels in double quotes: `A["My Label"]`
3. ALWAYS wrap edge labels in double quotes: `A -->|"label"| B`
4. No special characters in labels without quoting
5. Never use `end` as node ID (reserved) — use `End`/`finish`/`done`
6. Every `alt`/`opt`/`loop`/`par` needs matching `end`
7. State blocks with `{ }` need closing brace
8. No emoji/unicode in node text
9. `-->` solid, `-.->` dotted, `==>` thick
10. Subgraphs: `subgraph title` ... `end`
11. Use `<br/>` for line breaks (not `\n`)

## Inputs

- `--scope=<path>`, `--type=<flowchart|sequence|class|er|state>`, `--markers-only`, `--dry-run`
- Analysis data from `docs/ARCHITECTURE.md`, `docs/API-SURFACE.md`, `docs/DEPENDENCY-GRAPH.md`

**Prerequisite**: `docs/ARCHITECTURE.md` must exist (run `doc-analyzer` first).

## Pipeline

### Step 1: Read Analysis Data

Parse `docs/ARCHITECTURE.md`, `docs/API-SURFACE.md`, `docs/DEPENDENCY-GRAPH.md`, `docs/MODULE-SUMMARIES.md`. Extract module names, dependencies, public types, entry points, state-like enums.

### Step 2: Collect Diagram Requests

**From markers**: Scan `docs/**/*.md` for `<!-- DIAGRAM: type=X scope=Y title="Z" -->`.
**From manifest**: Read `## Diagram Manifest` in `docs/ARCHITECTURE.md`.
**From custom registry**: Read `docs/diagrams/CUSTOM.md` if exists.
**From auto-detection** (unless `--markers-only`): 3+ cross-importing modules → flowchart, CLI→service→storage → data flow, 3+ types with inheritance → class diagram, multi-step cross-module flow → sequence, enum with 3+ states → state machine.

### Step 3: Deduplicate

Priority: Markers > Custom registry > Manifest > Auto-detection.

### Step 4: Generate

Per request: read source files, build Mermaid syntax per skill patterns. Constraints: max 15 nodes/flowchart, max 10 participants/sequence, subgraphs at 8+ nodes, labels < 40 chars.

### Step 5: Validate & Fix

1. Quick check: valid diagram type, balanced brackets
2. If `mmdc` available: `echo '<code>' | mmdc -i /dev/stdin -o /tmp/check.svg --quiet 2>&1`
3. On error: fix exact issue, re-validate. Max 3 retries.
4. If still failing: write best version with `<!-- VALIDATION WARNING: ... -->`.

### Step 6: Write Output

**Marker-based**: Insert between `DIAGRAM-START`/`DIAGRAM-END` fences (idempotent replace).
**Manifest/auto**: Write to `docs/diagrams/<id>.md` with generated header, title, diagram, key, related links.

### Step 7: Build Index

Write/update `docs/diagrams/INDEX.md` cataloging all diagrams (auto-generated + manual). Never modify manual diagrams.

### Step 8: Report

Print summary: counts by source (markers, custom, manifest, auto-detected), files written, inline embedded.

## Constraints

- Never modify files without `<!-- Generated by diagram-generator -->` header (unless in CUSTOM.md)
- Never modify content outside DIAGRAM-START/DIAGRAM-END fences
- Prefer clarity over completeness
- Use actual module/function names, not placeholders
